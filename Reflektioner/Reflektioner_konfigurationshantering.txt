3. Kort sammanfattning om vad konfigurationshantering är och varför det
används.

Det används för att säkerställa att alla utvecklare arbetar med samma version av programvaran, spåra vad som ändras, och snabbt kunna återskapa tidigare versioner om något går fel. Det hjälper också att automatisera och standardisera systeminställningar, vilket gör det enklare att hantera stora projekt och undvika misstag.

4. Kort sammanfattning om det vanligaste arbetsflödet med git, inklusive de
git-kommandon som används.

Det vanligaste arbetsflödet med Git ser ungefär ut så här:

Klona ett repo: Kopiera ett existerande repository från GitHub till din lokala dator.
git clone <repo-url>

Skapa en ny gren: Arbeta i en separat gren för att hålla ändringarna isolerade från huvudkoden.
git checkout -b <ny-gren>

Gör ändringar: Ändra filer i projektet.

Staga ändringar: Förbered ändrade filer för att sparas i Git.
git add <filnamn> eller git add . för alla ändringar.

Spara ändringar (commit): Skapa en punkt i historiken med ett meddelande som beskriver vad du ändrat.
git commit -m "Beskrivning av ändringar"

Push till fjärr-repo: Skicka dina ändringar till det gemensamma repo (t.ex. på GitHub).
git push origin <ny-gren>

Skapa en Pull Request: När ändringarna är redo, öppna en Pull Request på GitHub för att få koden granskad och sammanslagen med huvudgrenen.
Hämta nya ändringar: Uppdatera din lokala kopia med eventuella ändringar från fjärr-repot.
git pull
